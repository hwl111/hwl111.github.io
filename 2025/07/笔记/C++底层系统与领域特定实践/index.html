C++底层系统与领域特定实践

C++ 底层、系统与领域特定实践

一、与系统交互

1. Linux/Windows系统编程

POSIX API (Linux)

WinAPI (Windows)

2. 文件I/O高级操作

mmap (Linux)

io_uring (Linux, 5.1+)

  • 需liburing库，适合高性能异步I/O，详见liburing官方文档。

3. 网络编程

socket (POSIX)

Boost.Asio

libuv (C风格)

  • 适合高性能事件驱动网络，详见libuv官方文档。

4. 进程间通信 (IPC)

pipe

shared memory

message queue

二、嵌入式与实时系统

1. 资源受限环境C++技巧

  • 禁用RTTI: -fno-rtti，禁用异常: -fno-exceptions。
  • 用自定义分配器、对象池，避免动态分配。

2. 实时性保障与C++特性选择

  • 避免不确定性操作（如new、虚函数、异常）。
  • 用constexpr、模板元编程提升编译期计算。

3. 与硬件寄存器交互

  • 内存映射I/O常用于驱动开发。

三、游戏开发

1. 游戏引擎架构中的C++应用

ECS模式（实体-组件-系统）

2. 资源管理与脚本绑定

  • 用智能指针、资源池管理纹理/模型。
  • 脚本绑定：pybind11/Lua/C#等。

3. 游戏物理、图形、AI算法实现

  • 物理：刚体动力学、碰撞检测（AABB、BVH）。
  • 图形：OpenGL/Vulkan渲染管线。
  • AI：A*寻路、行为树。

4. 实时渲染优化技巧

  • 批量绘制、GPU实例化、延迟渲染。

四、金融与高频交易

1. 极致低延迟编程

  • cache line对齐：alignas(64) struct Data { ... }。
  • 避免分支、用无锁结构（如环形缓冲区）。

案例：无锁环形缓冲区

2. 内存访问模式优化

  • 顺序访问、预取、避免false sharing。

五、与其他语言交互

1. C接口（extern “C”）

2. Python绑定（pybind11）

3. Rust FFI
